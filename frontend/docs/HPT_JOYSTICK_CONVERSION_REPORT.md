# 조이스틱 Rg↔hp 변환 계수 분석 보고서

> **최종 수정 (3/5 통일)**: 타점 한계선 = 공 반지름의 3/5 (0.858 Rg), 내부 ±4와 동일 물리 기준.

## 1. 변환 계수의 의미

`RG_TO_TIP_SCALE = 4 / MAX_HP_RADIUS_RG` (≈ 4.663, MAX_HP_RADIUS_RG = 0.858)

- **Rg 좌표계 → 내부 hpX/hpY 좌표계**로의 **선형 스케일 변환**
- Rg 단위 거리 0.858(3/5R)을 내부 단위 4에 맞춤

```
hpX = rgX × (4 / 0.858)
hpY = rgY × (4 / 0.858)
```

---

## 2. MAX_HP_RADIUS_RG 값의 산출 근거 (3/5 통일 후)

```
MAX_HP_RADIUS_RG = BALL_RADIUS_RG × TIP_LIMIT_RATIO
                 = 1.43 × (3/5)
                 ≈ 0.858
```

- **BALL_RADIUS_RG** = 1.43 (당구공 반지름, Rg 단위)
- **TIP_LIMIT_RATIO** = 3/5 (타점 한계선 = 공 반지름의 60%)
- **MAX_HP_RADIUS_RG** = 1.43 × 0.6 = **0.858** Rg

**코드 출처**: `useHptController.ts`

```ts
export const BALL_RADIUS_RG = 1.43;
export const TIP_LIMIT_RATIO = 3 / 5;
export const MAX_HP_RADIUS_RG = BALL_RADIUS_RG * TIP_LIMIT_RATIO; // 0.858
export const RG_TO_TIP_SCALE = 4 / MAX_HP_RADIUS_RG;
```

---

## 3. 물리 기준 통일 (3/5)

**TIP 모드, SPIN 모드, 조이스틱, 타점 시각화가 모두 동일한 물리 기준(3/5R)을 사용.**

- 타점 한계선 = 공 반지름의 3/5 = 0.858 Rg
- 내부 좌표 ±4 = 0.858 Rg에 정규화
- **Rg 0.858 = 내부 좌표 4**

---

## 4. 단순 스케일 보정 vs 곡률 보정

**단순 선형 스케일 보정이다.** 곡률 보정은 없다.

- 비율: `k = 4 / 1.287`
- 변환: `(hpX, hpY) = (rgX × k, rgY × k)`
- 각도·곡률은 변경하지 않고, 거리만 확대

---

## 5. 이 값을 제거하면?

변환을 제거하고 Rg를 그대로 hp로 쓴다면:

| 항목 | 변환 있음 (현재) | 변환 제거 |
|------|------------------|-----------|
| 조이스틱 가장자리 Rg | 1.287 | 1.287 |
| hpX, hpY 최대값 | 4.0 | **1.287** |
| 사용 가능 범위 | ±4 | **±1.287** |
| 체감 | 가장자리까지 ±4 | 가장자리까지 1.287 |

- 조이스틱 끝까지 움직여도 hp는 최대 ±1.287
- useHptController의 ±4 클램프 안으로 들어감
- TIP 모드 외곽(radius=4)과 크게 어긋남
- **결과**: 조이스틱이 “압축된” 느낌, 물리 반경과 내부 좌표 간 불일치

---

## 6. "타점값→회전값 변환 잔여 로직" 여부

**아니다.** 이 로직은 **좌표계 정규화**이다.

- **Rg**: 당구공 중심으로부터의 물리적 거리 (최대 1.287)
- **hpX, hpY**: 내부 표준 좌표 (범위 ±4)

둘 다 SPIN 모드에서 **회전/당점**에 쓰이는 동일한 직교 좌표이며,  
TIP 모드의 시계각(tip)과 회전값을 서로 바꾸는 변환이 아니다.

즉, **물리 Rg 거리 → 내부 ±4 좌표**로 맞추는 역할만 한다.

## 7. 요청 항목 정리 (3/5 통일 후)

### 7-1. 조이스틱 원 반지름 (픽셀)

| 구분 | 픽셀 |
|------|------|
| **바깥 테두리 반지름** | 100 px (`JOYSTICK_SIZE/2 = 200/2`) |
| **3/5R 한계선 반지름** | 60 px (`MAX_HP_RADIUS_RG × SCALE`) |

- `SCALE = 200 / (2×1.43) ≈ 69.93` px/Rg

### 7-2. Rg 좌표 최대값

- **최대 절대값**: 0.858 Rg (3/5 × 1.43)

### 7-3. 변환 전/후 좌표 예시

| 조이스틱 Rg 출력 | RG_TO_TIP_SCALE (×4.663) | hpX / hpY |
|------------------|---------------------------|-----------|
| (0, 0) | (0, 0) | (0, 0) |
| (0.429, 0) | (2.0, 0) | (2.0, 0) |
| (0.858, 0) | (4.0, 0) | (4.0, 0) |
| (0, -0.858) | (0, -4.0) | (0, -4.0) |
| (0.5, 0.5) | (2.33, 2.33) | (2.33, 2.33) |

### 7-4. hpX, hpY 최종값 예시

조이스틱이 (0.858, 0) Rg를 내보내면:

```
rgX = 0.858,  rgY = 0
hpX = 0.858 × (4/0.858) = 4.0
hpY = 0
→ setJoystick(4.0, 0)
→ hpX = 4.0, hpY = 0
```
